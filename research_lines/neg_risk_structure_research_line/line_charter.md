# neg_risk_structure_research_line
# polyarb_lab — research_line (active)

## Purpose

Investigate the structural pricing properties of Polymarket neg-risk markets.
Neg risk is an official Polymarket market structure: a No share in one outcome can be converted
into Yes shares in all other outcomes via the official neg-risk CTF adapter contract.

This line is NOT for:
- live trading
- claiming profitability
- contaminating mainline
- promoting ideas without repeated evidence
- mixing with other strategy families

## Background

Reward/rebate-aware market making has been tested locally and is not profitable so far.
That line is downgraded. This line is the new research priority.

Neg risk is not a community rumor. Official references:
- Polymarket neg risk docs (docs.polymarket.com)
- Polymarket/neg-risk-ctf-adapter (official adapter contract and repo)
- Polymarket/py-clob-client (official Python CLOB client)
- Polymarket/clob-client (official TypeScript CLOB client)
- PaulieB14/polymarket-subgraph-analytics (subgraph reference)
- NYTEMODEONLY/polyterm (terminal reference tool)

## Structural Claim Under Research

In a neg-risk market with N outcomes, the sum of all outcome YES prices must satisfy:
    sum(P_yes_i for i in 1..N) = 1.0

Deviation from this constraint (implied_sum != 1.0 with margin to clear spread + fee) is the
structural inconsistency this line investigates. It is NOT assumed to be profitable.
It is assumed to exist. Profitability depends on execution quality.

## Module Architecture

Module 1 — discovery.py
    Discover live neg-risk events via Gamma API.
    Output: list of raw neg-risk events with all associated markets.

Module 2 — normalizer.py
    Convert multi-outcome yes/no markets into a unified structural representation.
    Output: NegRiskEvent dataclass with implied_sum, constraint_gap, per-outcome prices.

Module 3 — structural_check.py
    Detect pricing inconsistencies in normalized events.
    Classify each event: CONSTRAINT_VIOLATION / BOUNDARY / AT_CONSTRAINT / UNKNOWN.
    Output: StructuralCheckResult with gap magnitude, direction, per-leg details.

Module 4 — executable_filter.py
    Apply execution quality gates: spread, depth, slippage, liquidity quality.
    Output: ExecutionQuality enum — EXECUTABLE / RESEARCH_VALUE_ONLY / NOISE.

Module 5 — paper_logger.py
    Log all scan results to data/research/neg_risk/ as JSON.
    Never submits orders. Never touches live execution.

## Isolation Rules

- No output from this line enters mainline without:
  1. Repeated structural evidence across >= 5 independent scans
  2. Execution quality confirmed EXECUTABLE in >= 3 scans at same event
  3. Explicit promotion decision logged in line notes
- All data written to data/research/neg_risk/ only (never data/processed/ or data/raw/)
- No imports from src/ in this line's modules
- No order submission of any kind

## Evidence Standards

- structural fact: confirmed in >= 3 independent scans on different events
- constraint gap: measured on CLOB-fetched prices, not Gamma outcomePrices alone
- execution quality: must use actual book depth and spread, not notional estimates
- escalation: gap must persist for >= 5 consecutive scans on the same event

## Classification Options for Hypotheses

- keep: active research value, continue testing
- downgrade: low value but not zero; reduce priority
- reject: no evidence of value; stop
- park: indeterminate; revisit when conditions change
- escalate: repeated evidence, promotion candidate
